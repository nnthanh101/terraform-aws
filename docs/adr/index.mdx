## ADR Process

### Naming Convention

Files follow kebab-case with zero-padded three-digit sequence numbers (ADR-NNN) per ADR-001:

```
ADR-NNN-short-title-slug.md
```

### Lifecycle

| Status | Meaning |
|--------|---------|
| Proposed | Under review, not yet ratified |
| Accepted | Ratified and in effect |
| Superseded | Replaced by a later ADR; retained for governance audit trail |
| Deprecated | No longer applicable; retained for historical record |

### Format

Each ADR record in DevOps-TechDocs contains:

- **Context**: The situation and forces at play
- **Decision**: The choice made and the rationale
- **Consequences**: Trade-offs, follow-on actions, and affected components
- **Status**: Current lifecycle state with supersession references where applicable

### Adding a New ADR

1. Draft the ADR document in DevOps-TechDocs (canonical source)
2. Add a row to this index table with the correct sequence number, title, status, and absolute URL
3. Reference the ADR number in any related Taskfile targets, CHANGELOG entries, or CLAUDE.md notes
4. HITL review required before status moves from Proposed to Accepted
