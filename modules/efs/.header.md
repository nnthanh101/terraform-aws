# AWS EFS Terraform Module

> Derived from [terraform-aws-modules/terraform-aws-efs](https://github.com/terraform-aws-modules/terraform-aws-efs). See [NOTICE](NOTICE.txt).

## Features

- EFS file system with One Zone or Standard storage classes
- Performance modes: generalPurpose or maxIO
- Throughput modes: bursting, elastic, or provisioned
- KMS encryption at rest with custom CMK support
- Lifecycle policies for Infrequent Access (IA) and Archive tiers
- Mount target management across multiple AZs
- Security group creation with NFS (port 2049) rules
- Backup policy integration with AWS Backup
- Cross-region replication configuration
- File system and access point policies

## Basic Usage

```hcl
module "efs" {
  source = "github.com/nnthanh101/terraform-aws//modules/efs?ref=v2.0.0"

  name           = "my-efs"
  encrypted      = true
  kms_key_arn    = "arn:aws:kms:ap-southeast-2:123456789012:key/xxx"

  performance_mode = "generalPurpose"
  throughput_mode  = "elastic"

  lifecycle_policy = {
    transition_to_ia = "AFTER_30_DAYS"
  }

  mount_targets = {
    "ap-southeast-2a" = { subnet_id = "subnet-a" }
    "ap-southeast-2b" = { subnet_id = "subnet-b" }
  }

  security_group_rules = {
    vpc = {
      type        = "ingress"
      from_port   = 2049
      to_port     = 2049
      protocol    = "tcp"
      cidr_blocks = ["10.0.0.0/16"]
    }
  }

  tags = {
    Environment = "production"
  }
}
```
